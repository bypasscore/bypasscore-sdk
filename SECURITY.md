# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 4.x     | :white_check_mark: |
| 3.x     | :white_check_mark: |
| 2.x     | Security fixes only|
| < 2.0   | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability in the BypassCore SDK, please report
it responsibly.

**Do NOT open a public GitHub issue for security vulnerabilities.**

Instead, please email: **security@bypasscore.com**

We will acknowledge your report within 48 hours and provide a detailed response
within 7 days. We may ask for additional information or guidance.

## Responsible Use

The BypassCore SDK is designed for **security research, reverse engineering
education, and authorized testing only**. Users are expected to:

- Only use these tools on systems they own or have explicit authorization to test
- Comply with all applicable laws and regulations
- Not use this software to circumvent software protections for piracy
- Not use this software to develop malware or other harmful tools

## Disclosure Policy

We follow coordinated disclosure. When we receive a vulnerability report:

1. We confirm the vulnerability and determine its impact
2. We develop and test a fix
3. We release the fix and publish an advisory
4. We credit the reporter (unless they prefer anonymity)
